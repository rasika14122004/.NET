Activities
-Why MVC used in companies.
	Companies use the Model-View-Controller (MVC) architectural pattern to improve code organization, maintainability,
	and scalability. By separating applications into three distinct layers—Model (data), View (UI), and 
	Controller (logic)—companies facilitate parallel development, where developers can work on different components 
	simultaneously without interference, speeding up project timelines. 
-What is MVC folder structure.
	A standard Model-View-Controller (MVC) folder structure separates application code into three main folders—Models (data), 
	Views (UI), and Controllers (logic)—to ensure modularity. Key directories include /Controllers for request handling,
	/Views for templating, and /Models for data structures, typically complemented by /Content or /Public for static assets (CSS/JS).
-What is Routing in MVC.
	Routing in MVC is a pattern-matching system that maps incoming browser requests to specific controller action methods. 
	Instead of requiring every URL to correspond to a physical file (like an .aspx or .php file), routing allows you to define 
	descriptive, SEO-friendly URL patterns that the application logic handles internally.
